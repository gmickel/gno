# Allow empty content in POST /api/docs

## Description

## Summary

POST /api/docs (document creation) currently rejects empty content strings.

## Current Behavior

```typescript
// src/serve/routes/api.ts:682
if (\!body.content || typeof body.content \!== 'string') {
  return errorResponse('VALIDATION', 'Missing or invalid content');
}
```

The `\!body.content` check rejects empty string `''` because it's falsy.

## Proposed Change

```typescript
if (typeof body.content \!== 'string') {
  return errorResponse('VALIDATION', 'Missing or invalid content');
}
```

This matches PUT /api/docs/:id (update) which already allows empty content:

```typescript
// src/serve/routes/api.ts:545-546
if (typeof body.content \!== 'string') {
  return errorResponse('VALIDATION', 'Missing or invalid content');
}
```

## Use Case

Users may want to create placeholder files or empty notes for later filling in.
