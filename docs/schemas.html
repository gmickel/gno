Output Schemas

JSON Schema definitions for all GNO output formats. Use these to validate and parse GNO responses.

About Schemas

GNO uses JSON Schema Draft-07to define all output formats. Schemas are available in the repository at spec/output-schemas/.

‚ÑπÔ∏è

All schemas use the gno://schemas/namespace with version suffixes (e.g., gno://schemas/search-results@1.0).

Validation

Use any JSON Schema validator (like Ajv) to validate GNO output:

validation.ts 
import Ajv from 'ajv';
import schema from './spec/output-schemas/search-results.schema.json';

const ajv = new Ajv();
const validate = ajv.compile(schema);

const output = JSON.parse(gnoOutput);
if (validate(output)) {
  // Valid SearchResults
} else {
  console.error(validate.errors);
}

SearchResults

Response from gno searchcommand with multiple results.

SearchResults gno://schemas/search-results@1.0
  • results array<SearchResult> required

    Array of search result items

  • meta object required

    Search metadata

meta Properties

  • query string required

    Original search query

  • mode enum required

    Search mode: bm25, vector, hybrid, or bm25_only

  • totalResults integer required

    Total number of results returned

  • expanded boolean

    Whether query expansion was used

  • reranked boolean

    Whether reranking was applied

  • vectorsUsed boolean

    Whether vector search was used

  • collection string

    Collection filter if applied

Example

{
  "results": [
    {
      "docid": "#a1b2c3d",
      "score": 0.89,
      "uri": "gno://notes/auth/login.md",
      "snippet": "User authentication uses JWT...",
      "source": {
        "relPath": "auth/login.md",
        "mime": "text/markdown",
        "ext": ".md"
      }
    }
  ],
  "meta": {
    "query": "authentication",
    "mode": "hybrid",
    "totalResults": 1,
    "reranked": true,
    "vectorsUsed": true
  }
}

SearchResult

Individual search result item.

SearchResult gno://schemas/search-result@1.0
  • docid string required

    Document ID (short hash, pattern: #[a-f0-9] { 6,8})

  • score number required

    Relevance score (0-1 normalized)

  • uri string required

    Document URI (pattern: gno://[collection]/[path])

  • snippet string required

    Matching text snippet from document

  • source object required

    Source file metadata

  • title string

    Extracted document title

  • snippetRange object

    Line range of snippet (startLine, endLine)

  • context string

    Context metadata from collection

source Properties

  • relPath string required

    Path relative to collection root

  • mime string required

    MIME type of source file

  • ext string required

    File extension (including dot)

  • absPath string

    Absolute path (with --sourceflag)

  • modifiedAt string

    Last modification time (ISO 8601)

  • sizeBytes integer

    File size in bytes

AskResponse

Response from gno askcommand.

AskResponse gno://schemas/ask@1.0
  • query string required

    Original query

  • mode enum required

    Retrieval mode: hybridor bm25_only

  • results array<SearchResult> required

    Search results used for context

  • meta object required

    Retrieval metadata

  • answer string

    Grounded answer (with --answerflag)

  • citations array

    Citations for the answer

citations Item Properties

  • docid string required

    Document ID

  • uri string required

    Document URI

  • startLine integer

    Citation start line

  • endLine integer

    Citation end line

Example

{
  "query": "How does authentication work?",
  "mode": "hybrid",
  "answer": "Authentication uses JWT tokens stored...",
  "citations": [
    {
      "docid": "#a1b2c3d",
      "uri": "gno://notes/auth/overview.md",
      "startLine": 15,
      "endLine": 28
    }
  ],
  "results": [...],
  "meta": {
    "expanded": false,
    "reranked": true,
    "vectorsUsed": true,
    "answerGenerated": true
  }
}

GetResponse

Response from gno getcommand.

GetResponse gno://schemas/get@1.0
  • uri string required

    Document URI

  • docid string required

    Document ID

  • content string required

    Document content (mirror text)

  • source object required

    Source file metadata (same as SearchResult.source)

  • range object

    Line range if --lineswas used

MultiGetResponse

Response when retrieving multiple documents.

MultiGetResponse gno://schemas/multi-get@1.0
  • documents array<GetResponse> required

    Array of retrieved documents

  • errors array

    Any retrieval errors (URI + error message)

StatusResponse

Response from gno statuscommand.

StatusResponse gno://schemas/status@1.0
  • version string required

    GNO version

  • collections array required

    Array of collection status objects

  • totalDocuments integer required

    Total documents across all collections

  • totalSizeBytes integer required

    Total storage size in bytes

collections Item Properties

  • name string required

    Collection name

  • documentCount integer required

    Number of documents

  • sizeBytes integer required

    Collection size in bytes

  • lastIndexed string

    Last indexing time (ISO 8601)

Example

{
  "version": "0.1.0",
  "totalDocuments": 1247,
  "totalSizeBytes": 25690112,
  "collections": [
    {
      "name": "notes",
      "documentCount": 523,
      "sizeBytes": 12902400,
      "lastIndexed": "2024-01-15T10:30:00Z"
    },
    {
      "name": "code",
      "documentCount": 724,
      "sizeBytes": 12787712,
      "lastIndexed": "2024-01-15T09:15:00Z"
    }
  ]
}

ErrorResponse

Standard error response format.

ErrorResponse gno://schemas/error@1.0
  • error object required

    Error details

error Properties

  • code string required

    Error code (e.g., NOT_FOUND, INVALID_QUERY)

  • message string required

    Human-readable error message

  • details object

    Additional error context

Example

{
  "error": {
    "code": "NOT_FOUND",
    "message": "Document not found: #xyz123",
    "details": {
      "docid": "#xyz123"
    }
  }
}
