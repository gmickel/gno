CLI Reference

Complete reference for all GNO command-line interface commands, flags, and options.

gno add

Index files or directories into a collection for searching.

gno add <path> Indexing

Recursively indexes files from the specified path into a named collection. Files are converted to searchable text and stored in the local database.

Usage

gno add <path> [options]
Option Description Default
--collection, -c Collection name for grouping documents Directory name
--recursive, -r Recursively index subdirectories true
--watch, -w Watch for file changes and auto-reindex false
--include Glob patterns to include (comma-separated) *
--exclude Glob patterns to exclude node_modules,.*
--context Context metadata for the collection -

Examples

# Index a notes folder
gno add ~/Documents/notes --collection notes

# Index with custom patterns
gno add ./src --include "*.ts,*.tsx" -c code

# Watch for changes
gno add ~/projects/docs --watch -c docs

gno search

Search indexed documents using natural language queries.

gno search <query> Retrieval

Performs hybrid BM25 + vector search across indexed documents. Returns ranked results with relevance scores and text snippets.

Usage

gno search <query> [options]
Option Description Default
--collection, -c Filter to specific collection All collections
--limit, -n Maximum number of results 10
--mode, -m Search mode: hybrid, bm25, vector hybrid
--rerank Apply reranking to results true
--expand Use query expansion false
--source Include absolute paths in output false
--json Output as JSON false

Examples

# Basic search
gno search "authentication flow"

# Search specific collection with limit
gno search "API endpoints" -c docs -n 5

# BM25-only mode with JSON output
gno search "error handling" --mode bm25 --json

# Include source paths
gno search "database schema" --source

gno ask

Ask questions and get grounded answers with citations.

gno ask <question> Retrieval + Answer

Combines search with optional LLM-generated answers. Returns relevant documents and can synthesize answers grounded in your local knowledge.

Usage

gno ask <question> [options]
Option Description Default
--answer, -a Generate grounded answer (requires LLM config) false
--collection, -c Filter to specific collection All collections
--limit, -n Maximum context documents 5
--lang Query language hint (BCP-47) auto
--json Output as JSON false

Examples

# Get relevant documents
gno ask "How does the payment system work?"

# Get a synthesized answer with citations
gno ask "What are the API rate limits?" --answer

# JSON output for programmatic use
gno ask "deployment process" -a --json

gno get

Retrieve a specific document by URI or document ID.

gno get <uri|docid> Retrieval

Fetches the full content and metadata of a specific document. Supports both GNO URIs and short document IDs from search results.

Usage

gno get <uri|docid> [options]
Option Description Default
--lines Line range to retrieve (e.g., 10-50) Full document
--meta Include full metadata false
--json Output as JSON false

Examples

# Get by URI
gno get "gno://notes/auth/overview.md"

# Get by document ID from search
gno get "#a1b2c3"

# Get specific line range
gno get "#a1b2c3" --lines 15-30

# Get with full metadata
gno get "gno://code/src/auth.ts" --meta --json

gno status

Display index statistics and collection information.

gno status Info

Shows current index statistics including document counts, collection sizes, and storage usage.

Usage

gno status [options]
Option Description Default
--collection, -c Show status for specific collection All collections
--json Output as JSON false

Example Output

================

Total Documents: 1,247

Collections:
  notes     - 523 docs (12.3 MB)
  code      - 412 docs (8.7 MB)
  docs      - 312 docs (3.5 MB)

gno config

Manage GNO configuration settings.

gno config Configuration

View and modify GNO configuration including embedding models, LLM settings, and storage paths.

Usage

gno config [get|set|list] [key] [value]
Subcommand Description
list Show all configuration values
get <key> Get a specific configuration value
set <key> <value> Set a configuration value

Configuration Keys

Key Description Default
storage.path Path to database storage ~/.gno
embedding.model Embedding model identifier local
llm.provider LLM provider for answer generation none
llm.model LLM model identifier -
search.defaultLimit Default result limit 10

Examples

# List all config
gno config list

# Get a specific value
gno config get storage.path

# Set LLM provider
gno config set llm.provider anthropic
gno config set llm.model claude-3-haiku-20240307

gno mcp

Start the MCP server for AI assistant integration.

gno mcp Server

Launches the Model Context Protocol server over stdio. Designed for integration with Claude and other MCP-compatible AI assistants.

Global Flags

These flags are available for all commands.

Flag Description
--help, -h Show help for the command
--version, -v Show GNO version
--verbose Enable verbose output
--quiet, -q Suppress non-essential output
--config Path to config file

Exit Codes

GNO uses standard exit codes to indicate success or failure.

Code Meaning
0 Success
1 General error
2 Invalid arguments or usage
3 Resource not found (collection, document)
4 Configuration error
5 External service error (LLM, embedding)

Output Formats

GNO supports multiple output formats for different use cases.

Human-Readable (Default)

Formatted text output optimized for terminal display. Includes colors when supported.

JSON (--json)

Structured JSON output following the schemas defined in the Schemasreference. Ideal for scripting and integration.
