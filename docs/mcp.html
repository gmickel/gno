MCP Tools Reference

GNO provides a Model Context Protocol (MCP) server for seamless integration with Claude and other AI assistants.

Overview

The GNO MCP server exposes local knowledge search capabilities to AI assistants. It runs over stdio and follows the MCP specification.

‚ÑπÔ∏è

MCP (Model Context Protocol) is a standard for connecting AI assistants to external tools and data sources. GNO acts as an MCP server, providing search tools that Claude can use to query your local knowledge base.

Capabilities

  • Tools: Search, ask, get, and status operations
  • Resources: Direct document access via GNO URIs
  • Streaming: Real-time result streaming for large responses

Integration

Configure your AI assistant to use GNO as an MCP server.

Claude Desktop

Add GNO to your Claude Desktop configuration:

~/.config/claude/claude_desktop_config.json 
{
  "mcpServers": {
    "gno": {
      "command": "gno",
      "args": ["mcp"]
    }
  }
}

Claude Code

For Claude Code CLI integration:

~/.claude/settings.json 
{
  "mcpServers": {
    "gno": {
      "command": "gno",
      "args": ["mcp"]
    }
  }
}

After configuration, restart your AI assistant. GNO tools will be available for searching your indexed collections.

gno_search

Search indexed documents using natural language queries.

gno_search Tool

Performs hybrid BM25 + vector search across indexed documents. Returns ranked results with relevance scores and text snippets.

Input Schema

  • query string required

    Natural language search query

  • collection string

    Filter to a specific collection

  • limit integer

    Maximum number of results (default: 10)

  • mode enum

    Search mode: hybrid, bm25, or vector

Example

// Tool call
{
  "name": "gno_search",
  "arguments": {
    "query": "authentication middleware",
    "collection": "code",
    "limit": 5
  }
}

Response

Returns a SearchResultsobject with matching documents.

gno_ask

Ask questions and get grounded answers with citations.

gno_ask Tool

Combines search with optional LLM-generated answers. Returns relevant documents and can synthesize answers grounded in your local knowledge.

Input Schema

  • query string required

    Natural language question

  • collection string

    Filter to a specific collection

  • answer boolean

    Generate a grounded answer (requires LLM config)

  • limit integer

    Maximum context documents (default: 5)

Example

// Tool call
{
  "name": "gno_ask",
  "arguments": {
    "query": "How does the payment processing work?",
    "answer": true
  }
}

Response

Returns an AskResponseobject with results and optional answer.

gno_get

Retrieve a specific document by URI or document ID.

gno_get Tool

Fetches the full content and metadata of a specific document. Supports both GNO URIs and short document IDs.

Input Schema

  • uri string required

    Document URI (gno://...) or document ID (#abc123)

  • lines string

    Line range to retrieve (e.g., "10-50")

Example

// Tool call
{
  "name": "gno_get",
  "arguments": {
    "uri": "gno://notes/architecture/overview.md"
  }
}

Response

Returns a GetResponseobject with document content and metadata.

gno_status

Get index statistics and collection information.

gno_status Tool

Returns current index statistics including document counts, collection sizes, and available collections.

Input Schema

  • collection string

    Get status for a specific collection (optional)

Example

// Tool call
{
  "name": "gno_status",
  "arguments": {}
}

Response

Returns a StatusResponseobject with index statistics.

Available Resources

GNO exposes documents as MCP resources for direct access.

Resource URI Format

gno://{collection}/{relative-path}

Resource Templates

Template Description
gno:// { collection}/ { path} Access a specific document by path
gno:// { collection}/ List documents in a collection

Example Resource Access

// Read resource
{
  "method": "resources/read",
  "params": {
    "uri": "gno://notes/meeting-notes/2024-01-15.md"
  }
}

Versioning

The GNO MCP server follows semantic versioning.

Property Value
Protocol Version 2024-11-05
Server Version 0.1.0
Schema Version 1.0

Capability Negotiation

The server advertises these capabilities during initialization: 

{
  "capabilities": {
    "tools": {
      "listChanged": false
    },
    "resources": {
      "subscribe": false,
      "listChanged": false
    }
  }
}
